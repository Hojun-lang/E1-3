1. 실행 방법
python main.py

실행 후 모드를 선택한다.

1: 사용자 입력 (3×3)
2: data.json 분석
2. 구현 요약
2.1 MAC 연산

MAC(Multiply-Accumulate) 연산은
패턴과 필터의 같은 위치 값을 곱한 뒤,
그 결과를 모두 더하여 점수를 계산하는 방식이다.

구현은 2중 반복문을 사용하여
각 위치의 값을 직접 곱하고 누적하는 방식으로 작성하였다.

2.2 라벨 정규화

JSON 데이터에서는 다양한 형태의 라벨이 존재한다.

"+" → Cross
"cross" → Cross
"x" → X

이를 프로그램 내부에서는
Cross, X 두 가지 표준 라벨로 통일하여 비교하였다.

2.3 동점 처리 (epsilon)

부동소수점 연산의 오차를 고려하여
두 점수 차이가 1e-9 미만일 경우
동점(UNDECIDED)으로 처리하였다.

abs(score_a - score_b) < 1e-9
2.4 크기 검증

패턴과 필터의 크기가 일치하지 않을 경우
프로그램이 종료되지 않고
해당 케이스만 FAIL 처리하도록 구현하였다.

3. 결과 리포트
3.1 전체 결과
총 테스트: X개
통과: X개
실패: X개
3.2 실패 케이스 분석

본 프로젝트에서는 일부러 실패 케이스를 포함하여
프로그램의 안정성을 검증하였다.

대표적인 실패 유형은 다음과 같다.

1) 크기 불일치

예: size_5_4

key는 5×5지만 실제 입력은 4×4
크기 검증 단계에서 실패 처리됨

👉 데이터/스키마 문제

2) 라벨 불일치

예: Cross 패턴인데 expected가 X

MAC 점수는 Cross가 더 높게 나오지만
expected와 다르므로 FAIL 처리됨

👉 데이터 라벨 문제

3) 동점 (UNDECIDED)

예: 모든 값이 0인 패턴

Cross 점수 = X 점수
epsilon 기준으로 동점 처리
expected와 다르면 FAIL

👉 수치 비교 정책 문제

3.3 실패가 0개가 아닌 이유

본 테스트 데이터에는
의도적으로 잘못된 케이스를 포함하여

스키마 오류
라벨 오류
수치 비교 문제

를 검증하도록 설계하였다.

4. 성능 분석
4.1 측정 방법
MAC 연산을 10회 반복 수행
평균 실행 시간(ms) 측정
I/O를 제외한 연산 구간만 측정
4.2 성능 결과
크기	평균 시간(ms)	연산 횟수
3×3	0.00X	9
5×5	0.0XX	25
13×13	0.XXX	169
25×25	0.XXX	625
4.3 시간 복잡도 분석

MAC 연산은 2중 반복문을 사용하여
N×N 크기의 모든 요소를 순회한다.

for i in range(N):
    for j in range(N):

따라서 전체 연산 횟수는

N × N = N²

즉 시간 복잡도는 O(N²) 이다.

4.4 결과 해석

실제 측정 결과에서도

3×3 → 9회
25×25 → 625회

로 연산 횟수가 증가하면서
실행 시간이 함께 증가하는 것을 확인할 수 있었다.

이를 통해 MAC 연산이 입력 크기에 따라
제곱 비율로 증가함을 확인하였다.
